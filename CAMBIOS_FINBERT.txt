🔄 CAMBIOS EN EL CÓDIGO

ANTES (Bedrock):
───────────────────────────────────────────────────────────────────────────────
def analyze_sentiment(headline):
    response = bedrock_client.invoke_model(
        modelId='anthropic.claude-3-haiku-20240307-v1:0',
        body=json.dumps({
            'max_tokens': 256,
            'system': 'You are a financial analyst...',
            'messages': [{'role': 'user', 'content': f'Analyze: {headline}'}]
        })
    )
    return json.loads(response['body'].read())

# Latencia: 80-150ms por headline
# Costo: $0.0003 por headline (3 centavos por 100)
───────────────────────────────────────────────────────────────────────────────

DESPUÉS (FinBERT):
───────────────────────────────────────────────────────────────────────────────
# Cargar modelo UNA SOLA VEZ al iniciar Lambda
from transformers import AutoTokenizer, AutoModelForSequenceClassification
model = AutoModelForSequenceClassification.from_pretrained("ProsusAI/finbert")
tokenizer = AutoTokenizer.from_pretrained("ProsusAI/finbert")

def analyze_sentiment_batch(headlines):
    # Procesar TODOS los headlines en batch (mucho más rápido)
    inputs = tokenizer(headlines, padding=True, truncation=True)
    with torch.no_grad():
        outputs = model(**inputs)
    probabilities = torch.softmax(outputs.logits, dim=-1)
    
    # Extraer sentimientos y confianzas
    results = []
    for i, headline in enumerate(headlines):
        predicted_class = torch.argmax(probabilities[i]).item()
        confidence = float(probabilities[i][predicted_class])
        sentiment = {0: 'bearish', 1: 'neutral', 2: 'bullish'}[predicted_class]
        results.append({'headline': headline, 'sentiment': sentiment, 'confidence': confidence})
    
    return results

# Latencia: 15-30ms POR BATCH (no por headline)
# Para 1000 headlines: 15-30 segundos total (vs 80-150 segundos)
# Costo: $0 (solo Lambda compute)
───────────────────────────────────────────────────────────────────────────────

DIFERENCIAS CLAVE:
1. ✅ Batch processing: múltiples headlines en paralelo
2. ✅ Sin API calls: inferencia 100% local
3. ✅ Modelo especializado: FinBERT entrenado en finanzas
4. ✅ Caching: modelo cargado UNA SOLA VEZ en memory
