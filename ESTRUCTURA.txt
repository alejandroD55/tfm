TFM - SISTEMA DE TRADING CON AWS LAMBDA
========================================

ESTRUCTURA DEL PROYECTO:
========================

tfm/
├── 📄 README.md                    - Documentación completa del sistema
├── 📄 QUICK_START.md               - Guía rápida de configuración
├── 📄 ESTRUCTURA.txt               - Este archivo
│
├── 🔧 CONFIGURACIÓN
├── ├── etf_universe.json           - Lista de ETFs a analizar (SPY, QQQ, etc.)
├── ├── database_schema.sql         - Schema de Aurora PostgreSQL
├── ├── iam_policy.json            - Política IAM para las lambdas
├── ├── stepfunctions_definition.json - Orquestación del pipeline
├── └── deploy.sh                   - Script de deployment automático
│
└── 🔗 FUNCIONES LAMBDA (5 lambdas)
    │
    ├── lambda_ingestion/           [λ1 - INGESTA]
    │   ├── lambda_ingestion.py     - Lee config ETFs, descarga datos OHLCV, noticias
    │   └── requirements.txt        - Dependencies: yfinance, requests, psycopg2-binary, pandas
    │
    ├── lambda_sentiment/           [λ2 - SENTIMIENTO]
    │   ├── lambda_sentiment.py     - Analiza sentimiento usando FinBERT vía Hugging Face API
    │   └── requirements.txt        - Dependencies: psycopg2-binary, huggingface_hub
    │
    ├── lambda_indicators/          [λ3 - INDICADORES TÉCNICOS]
    │   ├── lambda_indicators.py    - Calcula RSI, SMA, Bandas de Bollinger
    │   └── requirements.txt        - Dependencies: psycopg2-binary, pandas, pandas-ta-classic
    │
    ├── lambda_bayesian/            [λ4 - INFERENCIA BAYESIANA]
    │   ├── lambda_bayesian.py      - Red bayesiana para señales de trading
    │   └── requirements.txt        - Dependencies: psycopg2-binary, pgmpy, numpy, scipy, pandas, networkx
    │
    └── lambda_report/              [λ5 - REPORTE]
        ├── lambda_report.py        - Backtesting y generación de reportes
        └── requirements.txt        - Dependencies: psycopg2-binary, pandas, numpy

MÉTRICAS CALCULADAS:
====================

Sentimiento:
  • Sentimiento: bullish, bearish, neutral
  • Confidence: 0.0 - 1.0
  • Justificación: texto

Indicadores Técnicos:
  • RSI 14:        0 - 100 (oversold < 30, overbought > 70)
  • SMA 20:        Precio medio 20 períodos
  • SMA 50:        Precio medio 50 períodos
  • BB Upper:      Banda superior Bollinger
  • BB Middle:     Banda media Bollinger
  • BB Lower:      Banda inferior Bollinger

Señales de Trading:
  • Signal:        BUY, SELL, HOLD
  • Prob_up:       P(MarketDirection=up)
  • Prob_down:     P(MarketDirection=down)

Backtesting:
  • Cumulative Return:      Retorno acumulado (%)
  • Sharpe Ratio:           Retorno ajustado por riesgo
  • Max Drawdown:           Peor caída capital (%)


CONFIGURACIÓN RECOMENDADA:
==========================

Lambda Memory:
  • lambda_ingestion:    512 MB
  • lambda_sentiment:    512 MB
  • lambda_indicators:   512 MB
  • lambda_bayesian:     1024 MB
  • lambda_report:       512 MB

Lambda Timeout:
  • Todos: 300 segundos (5 minutos)

Aurora Instance:
  • Mínimo: db.t4.small / Free Tier PostgreSQL
  • Recomendado: db.t4.medium o superior

Scheduling:
  • Daily execution a las 9:00 AM UTC
  • Via EventBridge cron rule: "0 9 * * ? *"


SEGURIDAD:
===========

✓ Credenciales en AWS Secrets Manager
✓ Política IAM con permisos mínimos
✓ DB en VPC privada (o securizada por Security Group)
✓ Encryption at rest para S3 y RDS
✓ Logs en CloudWatch con encriptación

No hardcodear:
✗ Credenciales en código
✗ API keys en variables de ambiente
✗ Passwords en configuración
